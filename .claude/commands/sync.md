実装完了後のメモリ・ドキュメント・draft 同期を行う。

## 手順

### 1. 直近の作業内容を把握

```bash
git log --oneline -10
git diff HEAD~1..HEAD --stat
```

直近のコミット履歴から「何が完了したか」「次に何をすべきか」を把握する。

### 2. MEMORY.md を更新

`/home/vscode/.claude/projects/-workspace/memory/MEMORY.md` を読んで、現在の状態と合っていない箇所を修正する。

具体的に確認・更新する項目:
- **完了済みステップ** — 実装が終わったものに `✅` や完了マークを付ける
- **次のステップ** — 完了したステップを外し、次のステップを正確に記す
- **コミットハッシュ** — 最新コミットのハッシュに更新する
- **古くなった情報** — 実装済みにより不要になったメモは削除する

不要になった詳細情報はメモリに残さず削除すること（MEMORY.md は 200 行以内を維持）。

### 3. draft ファイルを更新

`/workspace/.claude/drafts/` 内の関連ファイルを読んで、完了タスクのチェックボックスを更新する。

- `[ ]` → `[X]` に変更（完了済みの項目）
- コミットハッシュやブランチ名など事実情報を最新に修正
- 完了したステップには `✅ 完了（コミット: <hash>）` のようなメモを追記

### 4. ROADMAP.md（必要なら）

`/workspace/ROADMAP.md` が今回の実装と関連する milestone を含む場合のみ更新する。
関係ない場合はスキップ。

### 5. 更新内容をサマリーとして報告

更新した箇所の一覧をユーザーに報告する。形式:

```
### 更新済み
- MEMORY.md: <変更点>
- drafts/rust-migration.md: <変更点>
- （更新したファイルのみ列挙）
```
