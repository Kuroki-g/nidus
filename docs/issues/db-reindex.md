# nidus update — 既存インデックス全体の再構築

## 概要

インデックス済みのファイル全体を再エンベディングし直す `nidus update` コマンドを追加する。

## 背景

現在の `nidus add` は差分追加のみ。
エンベディングモデルを変更した場合や DB スキーマを更新した場合に、
全チャンクを作り直す手段がない。

## 実装方針

- 既存の `source` 一覧を取得し、全ファイルを `nidus add` 相当で再処理
- オプション: `--dry-run`（対象ファイル一覧のみ表示）
- テーブルを DROP & 再作成 か、上書き更新（upsert）かは要検討
