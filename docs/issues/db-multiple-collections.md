# 複数コレクション（名前付き DB）のサポート

## 概要

現在は単一テーブル（`TABLE_NAME` 固定）だが、用途別に名前付きコレクションを使い分けられるようにする。

## ユースケース

- 仕事用 / 個人用でインデックスを分離
- プロジェクト別に検索スコープを切り替える
- `nidus search "キーワード" --collection work`

## 実装方針

- `--collection <name>` オプションを各コマンドに追加
- `Settings.TABLE_NAME` をデフォルト値として維持し、オプション指定時に上書き
- MCP ツールにも `collection` パラメータを追加
